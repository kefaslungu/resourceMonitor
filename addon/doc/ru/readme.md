# Resource Monitor

* Авторы: Alex Hall, Joseph Lee, Kefas Lungu, Beqa Gozalishvili, Tuukka Ojala, Ethin Probst и другие участники NVDA

Это дополнение предоставляет информацию о загрузке процессора, используемой памяти и  других используемых ресурсах.

## Горячие клавиши

Все команды поддерживают режим речи по требованию.

* NVDA+Shift+E: предоставляет используемую оперативную память (физическую память) и среднюю загрузку процессора.
* NVDA+Shift+1: Предоставляет среднюю загрузку процессора и загрузку каждого ядра.
* NVDA+Shift+2/5: отображает использованный и общий объём как для физической, так и для виртуальной оперативной памяти (NVDA+Shift+5 является альтернативой NVDA+Shift+2, когда последняя комбинация клавиш не может быть выполнена).
* NVDA+Shift+3: presents the used and total space of the fixed (built-in), removable, and network drives.
* NVDA+Shift+4: предоставляет информацию о беспроводном соединении, имени и мощности сети (SSID) или об отсутствии SSID, если он недоступен.
* NVDA+Shift+6: предоставляет версию Windows, архитектуру ЦП и точный номер сборки (build.revision).
* NVDA+Shift+7: предоставляет время безотказной работы системы.
* Не назначена: предоставляет информацию о графическом процессоре (GPU; недоступно в безопасном режиме).
* Unassigned: presents graphics processing unit (GPU memory usage information; unavailable in secure mode).

Вы можете изменить эти сочетания клавиш в диалоге жестов ввода.

## Замечания по использованию

Это дополнение не заменяет диспетчер задач и другие информационные программы системы Windows. Также обратите внимание на следующее:

* Помимо команды общего использования ресурсов (NVDA+Shift+E), двойное нажатие других команд скопирует информацию об использовании ресурсов в буфер обмена.
* Информацию о ресурсах невозможно скопировать в буфер обмена при запуске дополнения на защищённых экранах.
* Использование процессора предоставляется для логических процессоров, а не физических ядер. Это касается процессоров, которые использует Hyper Threading, где количество процессоров в два раза превышает количество процессорных ядер. На некоторых новых компьютерах не на всех ядрах процессора будет включена поддержка hyper-threading.
* If there is heavy disk activity such as copying large files or while locating network drives, there might be delays when obtaining disk usage information.
* GPU information is given for Nvidia GPU's.
* При объявлении информации об архитектуре процессора "x86" и "AMD64" относятся к 32-разрядным и 64-разрядным (x64) процессорам Intel и AMD соответственно.
* Установка дополнения в Windows 10/11 LTSC не поддерживается.

Список изменений, вносимых между выпусками каждого дополнения, см. в документе [списки изменений для выпусков дополнения][1].
